<task type="auto">
  <name>Task 2: Create rotation service with daily rotation algorithm</name>
  <files>src/services/rotation.js</files>
  <action>
Create new file `src/services/rotation.js` with rotation logic:

1. `rotateCategoryPremiums(category)` - Rotate premium listings within a single category:
   - Get all premium listings in category ordered by rotation_position ASC
   - If 0 or 1 premium listings, no rotation needed (return early)
   - Take first listing (position 1), move to end: set its rotation_position to max+1
   - Renumber all others sequentially from 1 (so positions stay 1,2,3... not 1,3,5...)
   - Update last_rotated_at to now() for the listing that moved
   - Return { rotated: true, movedListing: name } or { rotated: false, reason: '...' }

2. `rotateAllCategories()` - Rotate all categories:
   - Get all distinct categories with premium listings
   - Call rotateCategoryPremiums for each
   - Return summary { categoriesProcessed: N, rotations: [...results] }

3. `needsRotation()` - Check if rotation should run:
   - Query any premium listing's last_rotated_at
   - If null or date part < today, return true
   - Else return false
   - This allows rotation to run once per day

Algorithm is simple "move first to last" - fair because each day a different business is first. Over N days, each of N premium members gets 1 day at top.

Do NOT add any scheduling/cron - that's Phase 4's concern (API can trigger rotation check on each request if needed, or we add cron later).
  </action>

  <done>Rotation service exports rotateCategoryPremiums, rotateAllCategories, needsRotation functions</done>
</task>

<task type="auto">

  <action>
Add method `getSortedByCategory(category)` to listing repository:

1. Query all listings in the category
2. Sort results: premium listings first (ordered by rotation_position ASC), then non-premium (ordered by name ASC for consistency)
3. Return the sorted array

This is the key method Phase 4's API endpoint will use. The sorting happens at query time, not in JS.

SQL approach:
```sql
SELECT * FROM listings
WHERE category = $1
ORDER BY is_premium DESC,
         CASE WHEN is_premium THEN rotation_position ELSE 999999 END ASC,
         name ASC
```

This ensures:
- Premium listings (is_premium=true) come first
- Premium listings sorted by rotation_position (1,2,3...)
- Non-premium listings sorted alphabetically by name
