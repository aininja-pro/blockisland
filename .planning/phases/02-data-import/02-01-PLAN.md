  <action>
Create import script that:
1. Reads JSON file path from command line argument
2. Parses GoodBarber maps JSON format (handles both single file and array of items)
3. Maps fields per DISCOVERY.md field mapping table:
   - id → goodbarber_id (as string)
   - title → name
   - subtype → category
   - content → description (keep HTML, strip later if needed)
   - address, phoneNumber→phone, website, email → direct
   - latitude/longitude → parse to float, handle string format
   - thumbnail OR largeThumbnail OR originalThumbnail → image_url (first available)
   - Default is_premium=false, rotation_position=0
4. Validates required fields (name, category)
5. Logs progress: total items, imported count, skipped with reasons
6. Uses listing repository for database operations

Handle edge cases:
- Missing coordinates: log warning, import with null lat/long
- Missing optional fields: use null
- Duplicate goodbarber_id: update existing record (upsert behavior)

Script should be idempotent - running twice produces same result.
