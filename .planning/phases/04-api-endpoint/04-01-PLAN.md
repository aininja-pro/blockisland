<task type="auto">
  <name>Task 1: Create feed router with GoodBarber transformation</name>
  <files>src/api/feed.js</files>
  <action>
Create Express router with:

1. GET /maps endpoint:
   - Check rotation with needsRotation(), run rotateAllCategories() if needed
   - Support optional ?category query param
   - If category provided: getSortedByCategory(category)
   - If no category: getAllCategories() then aggregate sorted results from each
   - Transform each listing to GoodBarber format
   - Return { items, next_page: null, stat: 'ok' }

2. transformToGoodBarber(listing) function:
   - id: parseInt(listing.goodbarber_id) or hashCode(listing.id) for numeric ID
   - title: listing.name
   - content: listing.description || ''
   - summary: strip HTML and truncate to 100 chars
   - address: listing.address || ''
   - latitude: String(listing.latitude || '')
   - longitude: String(listing.longitude || '')
   - phoneNumber: listing.phone || ''
   - email: listing.email || ''
   - website: listing.website || ''
   - date: listing.created_at
   - type: 'map'
   - subtype: listing.category
   - thumbnail: listing.image_url || ''
   - images: array with single image if image_url exists

3. hashCode(str) helper for UUID → integer conversion

AVOID: Don't add pagination (GoodBarber handles caching). Don't require authentication.
WHY: GoodBarber needs simple public endpoint, <100 listings per category doesn't need pagination.
