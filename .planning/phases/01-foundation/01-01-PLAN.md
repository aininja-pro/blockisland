**Tech stack decisions:**
- Express.js — Simple, well-documented API framework
- better-sqlite3 — Fast synchronous SQLite for Node.js (perfect for small backend)
- SQLite — Zero-config database, ideal for ~30-40 listings and 2-3 admins
- Deploy target: Render (~$10-20/month)

**Key constraints:**
- Must support lat/long coordinates for map pins
- Must support categories with independent rotation per category
- Premium rotation applies within each category
- ~30-40 premium members, premium status changes annually
</context>

<task type="auto">
  <name>Task 2: Create database schema and initialization</name>
  <files>src/db/schema.sql, src/db/init.js, data/listings.db</files>
  <action>
    Create schema.sql with listings table:
    - id (INTEGER PRIMARY KEY)
    - name (TEXT NOT NULL)
    - category (TEXT NOT NULL) — for per-category rotation
    - description (TEXT)
    - address (TEXT)
    - phone (TEXT)
    - website (TEXT)
    - email (TEXT)
    - latitude (REAL) — preserve for map pins
    - longitude (REAL) — preserve for map pins
    - image_url (TEXT)
    - is_premium (INTEGER DEFAULT 0) — boolean flag
    - rotation_position (INTEGER DEFAULT 0) — for daily rotation tracking
    - last_rotated_at (TEXT) — ISO timestamp of last rotation
    - goodbarber_id (TEXT) — original ID from GoodBarber for sync
    - created_at (TEXT DEFAULT CURRENT_TIMESTAMP)
    - updated_at (TEXT DEFAULT CURRENT_TIMESTAMP)

    Add index on category for efficient per-category queries.
    Add index on is_premium for filtering.

    Create init.js that:
    - Opens/creates database file at data/listings.db
    - Runs schema.sql if tables don't exist
    - Exports db instance for use in app
  </action>
  <verify>node src/db/init.js creates data/listings.db without errors, sqlite3 data/listings.db ".tables" shows listings table</verify>
  <done>Database file created, listings table exists with all columns and indexes</done>
</task>

<task type="auto">
  <name>Task 3: Create Listing repository with basic CRUD</name>
  <files>src/models/listing.js</files>
  <action>
    Create listing.js repository module with functions:
    - getAll() — returns all listings
    - getById(id) — returns single listing
    - getByCategory(category) — returns listings in category
    - create(data) — inserts new listing, returns created record
    - update(id, data) — updates listing, returns updated record
    - delete(id) — removes listing
