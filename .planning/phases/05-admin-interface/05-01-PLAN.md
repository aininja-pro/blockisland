- Separate Next.js app in `admin/` directory (not modifying Express backend)
- Admin talks directly to Supabase (same database as Express backend)
- Express backend only serves GoodBarber feed

**Tech stack for admin:**
- Next.js 15 (App Router)
- shadcn/ui (Tailwind + Radix primitives)
- Supabase client for auth + database
- TypeScript

**Supabase credentials:**
- Already configured in project root .env (SUPABASE_URL, SUPABASE_KEY)
- Admin will use same Supabase project

<task type="auto">
  <name>Task 1: Create Next.js app with shadcn/ui</name>
  <files>admin/ (new directory)</files>
  <action>
Create Next.js app in admin/ directory:

1. Run: `npx create-next-app@latest admin --typescript --tailwind --eslint --app --src-dir --import-alias "@/*"`
   - Choose defaults for all prompts

2. Initialize shadcn/ui:
   ```
   cd admin
   npx shadcn@latest init
   ```
   - Style: Default
   - Base color: Slate (professional look)
   - CSS variables: Yes

3. Add initial shadcn components needed for admin:
   ```
   npx shadcn@latest add button card input label table dialog dropdown-menu avatar badge separator sheet tabs form
   ```

4. Create admin/.env.local with Supabase credentials:
   ```
   NEXT_PUBLIC_SUPABASE_URL=<copy from root .env>
   NEXT_PUBLIC_SUPABASE_ANON_KEY=<copy from root .env>
   ```

5. Install Supabase client:
   ```
   npm install @supabase/supabase-js @supabase/ssr
   ```

AVOID: Don't use pages router (use App Router). Don't install extra UI libraries beyond shadcn.
WHY: App Router is the modern Next.js pattern. shadcn/ui provides all needed components.
  </action>
  <verify>cd admin && npm run dev starts without errors, localhost:3001 shows Next.js app</verify>
  <done>Next.js app runs, shadcn components available, Supabase client installed</done>
</task>
