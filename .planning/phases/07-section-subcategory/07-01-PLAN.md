  <files>src/db/schema.sql, src/db/migration-07-sections.sql</files>
  <action>
Create a new migration file `src/db/migration-07-sections.sql` that:

1. Add `section` column to listings table:
   ```sql
   ALTER TABLE listings ADD COLUMN section TEXT;
   ```

2. Create `subcategories` reference table:
   ```sql
   CREATE TABLE subcategories (
     id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
     section TEXT NOT NULL,
     name TEXT NOT NULL,
     display_order INTEGER DEFAULT 0,
     created_at TIMESTAMPTZ DEFAULT NOW(),
     UNIQUE(section, name)
   );
   CREATE INDEX idx_subcategories_section ON subcategories(section);
   ```

3. Create `listing_subcategories` junction table:
   ```sql
   CREATE TABLE listing_subcategories (
     listing_id UUID REFERENCES listings(id) ON DELETE CASCADE,
     subcategory_id UUID REFERENCES subcategories(id) ON DELETE CASCADE,
     PRIMARY KEY (listing_id, subcategory_id)
   );
   CREATE INDEX idx_listing_subcategories_listing ON listing_subcategories(listing_id);
   CREATE INDEX idx_listing_subcategories_subcategory ON listing_subcategories(subcategory_id);
   ```

4. Enable RLS on new tables (allow public read, authenticated full access):
   ```sql
   ALTER TABLE subcategories ENABLE ROW LEVEL SECURITY;
   CREATE POLICY "Allow public read" ON subcategories FOR SELECT USING (true);
   CREATE POLICY "Allow authenticated full access" ON subcategories FOR ALL USING (auth.role() = 'authenticated');

   ALTER TABLE listing_subcategories ENABLE ROW LEVEL SECURITY;
   CREATE POLICY "Allow public read" ON listing_subcategories FOR SELECT USING (true);
   CREATE POLICY "Allow authenticated full access" ON listing_subcategories FOR ALL USING (auth.role() = 'authenticated');
   ```

Also update `schema.sql` to include these tables for reference (as comments or additional CREATE statements for new installs).
